## <a name="eye-gaze-input-design-guidelines"></a><span data-ttu-id="b8eb7-124">目视输入设计准则</span><span class="sxs-lookup"><span data-stu-id="b8eb7-124">Eye-gaze input design guidelines</span></span>

<span data-ttu-id="b8eb7-125">构建充分利用快速移动目视目标的交互可能会很困难。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-125">Building an interaction that takes advantage of fast-moving eye targeting can be challenging.</span></span> <span data-ttu-id="b8eb7-126">本部分汇总了设计应用程序时要考虑的主要优点和难题。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-126">In this section, we summarize the key advantages and challenges to consider when designing your application.</span></span> 

### <a name="benefits-of-eye-gaze-input"></a><span data-ttu-id="b8eb7-127">眼睛凝视输入的好处</span><span class="sxs-lookup"><span data-stu-id="b8eb7-127">Benefits of eye-gaze input</span></span>

- <span data-ttu-id="b8eb7-128">**高速指向。**</span><span class="sxs-lookup"><span data-stu-id="b8eb7-128">**High speed pointing.**</span></span> <span data-ttu-id="b8eb7-129">眼睛眼部是人体中反应速度最快的眼部。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-129">The eye muscle is the fastest reacting muscle in the human body.</span></span> 

- <span data-ttu-id="b8eb7-130">**不费力。**</span><span class="sxs-lookup"><span data-stu-id="b8eb7-130">**Low effort.**</span></span> <span data-ttu-id="b8eb7-131">几乎没有任何身体动作。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-131">Barely any physical movements are necessary.</span></span> 

- <span data-ttu-id="b8eb7-132">**隐含性。**</span><span class="sxs-lookup"><span data-stu-id="b8eb7-132">**Implicitness.**</span></span> <span data-ttu-id="b8eb7-133">有关用户眼睛运动的信息通常被用户描述为"思维读取"，让系统知道用户计划参与的目标。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-133">Often described by users as "mind reading", information about a user's eye movements lets the system know which target the user plans to engage.</span></span> 

- <span data-ttu-id="b8eb7-134">**备选的输入通道。**</span><span class="sxs-lookup"><span data-stu-id="b8eb7-134">**Alternative input channel.**</span></span> <span data-ttu-id="b8eb7-135">眼睛凝视可以为用户提供强大的支持输入，以基于用户的手部协调，基于他们多年来的经验构建手部和语音输入。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-135">Eye-gaze can provide a powerful supporting input for hand and voice input building on years of experience from users based on their hand-eye coordination.</span></span>

- <span data-ttu-id="b8eb7-136">**视觉注意力。**</span><span class="sxs-lookup"><span data-stu-id="b8eb7-136">**Visual attention.**</span></span> <span data-ttu-id="b8eb7-137">另一个重要好处是能够推断出用户关注什么。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-137">Another important benefit is the possibility to infer what a user is paying attention to.</span></span> <span data-ttu-id="b8eb7-138">这可在各种应用程序领域提供帮助，包括更有效地评估不同的设计，以及帮助更智能的用户界面和增强的远程通信社交提示。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-138">This can help in various application areas ranging from more effectively evaluating different designs to aiding in smarter user interfaces and enhanced social cues for remote communication.</span></span>

<span data-ttu-id="b8eb7-139">简而言之，使用眼睛凝视作为输入可提供快速轻松的上下文输入信号。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-139">In a nutshell, using eye-gaze as an input offers a fast and effortless contextual input signal.</span></span> <span data-ttu-id="b8eb7-140">当与其他输入（如语音和手动输入）结合使用以确认用户的意图时，此功能非常强大。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-140">This is powerful when combined with other inputs such as *voice* and *manual* input to confirm the user's intent.</span></span>


### <a name="challenges-of-eye-gaze-as-an-input"></a><span data-ttu-id="b8eb7-141">作为输入的眼睛凝视的挑战</span><span class="sxs-lookup"><span data-stu-id="b8eb7-141">Challenges of eye-gaze as an input</span></span>

<span data-ttu-id="b8eb7-142">虽然眼睛凝视可用于创建令人满足的用户体验，让你感觉自己像超级用户，但必须知道哪一点不适合适当考虑这一点。这一点也很重要。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-142">While eye-gaze can be used to create satisfying user experiences, which make you feel like a superhero, it's also important to know what it isn't good at to appropriately account for this.</span></span> <span data-ttu-id="b8eb7-143">以下列表讨论了在使用 *眼睛* 凝视输入时要考虑的一些难题以及如何解决这些问题：</span><span class="sxs-lookup"><span data-stu-id="b8eb7-143">The following list discusses some *challenges* to consider and how to address them when working with eye-gaze input:</span></span> 

- <span data-ttu-id="b8eb7-144">**眼睛凝视是"始终打开"** 一打开眼睛的眼圈，眼睛就开始固定环境中的东西。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-144">**Your eye-gaze is "always on"** The moment you open your eye lids, your eyes start fixating on things in the environment.</span></span> <span data-ttu-id="b8eb7-145">响应你做出的每一个外观并意外发出操作，因为你查看了太长内容，将导致不满足体验。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-145">Reacting to every look you make and accidentally issuing actions, because you looked at something for too long, would result in an unsatisfying experience.</span></span>
<span data-ttu-id="b8eb7-146">建议将眼睛凝视与语音命令、手势、按钮单击或延长停留相结合，以触发目标 (有关详细信息，请参阅眼睛凝视和提交) 。 [](gaze-and-commit-eyes.md)</span><span class="sxs-lookup"><span data-stu-id="b8eb7-146">We recommend combining eye-gaze with a *voice command*, *hand gesture*, *button click* or extended dwell to trigger the selection of a target (for more information, see [eye-gaze and commit](gaze-and-commit-eyes.md)).</span></span>
<span data-ttu-id="b8eb7-147">此解决方案还允许一种模式，在此模式下，用户可以自由地四处查看，而不会因不由自主地触发某些内容而感到重负。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-147">This solution also allows for a mode in which the user can freely look around without being overwhelmed by involuntarily triggering something.</span></span> <span data-ttu-id="b8eb7-148">当查看目标时设计视觉对象和听觉反馈时，还应考虑此问题。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-148">This issue should also be considered when designing visual and auditory feedback when looking at a target.</span></span>
<span data-ttu-id="b8eb7-149">尝试不要使用户塞满即时弹出效果或悬停声音。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-149">Try not to overwhelm the user with immediate pop-out effects or hover sounds.</span></span> <span data-ttu-id="b8eb7-150">个很微妙为 key。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-150">Subtlety is key.</span></span> <span data-ttu-id="b8eb7-151">讨论 [设计建议](eye-gaze-interaction.md#design-recommendations)时，我们将在下面讨论一些最佳实践。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-151">We'll discuss some best practices for this further below when talking about [design recommendations](eye-gaze-interaction.md#design-recommendations).</span></span>

- <span data-ttu-id="b8eb7-152">**观察与控制** 假设您想要在墙壁上精确地伸直照片。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-152">**Observation vs. control** Imagine that you want to precisely straighten a photograph on your wall.</span></span> <span data-ttu-id="b8eb7-153">你会参照它的边框和四周，检查它是否平齐。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-153">You look at its borders and its surroundings to see if it aligns well.</span></span> <span data-ttu-id="b8eb7-154">现在，如果您想要使用眼睛作为输入来移动图片，则可以想象出如何实现此目的。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-154">Now imagine how you would do that when you want to use your eye-gaze as an input to move the picture.</span></span> <span data-ttu-id="b8eb7-155">有点难度，对不对？</span><span class="sxs-lookup"><span data-stu-id="b8eb7-155">Difficult, isn't it?</span></span> <span data-ttu-id="b8eb7-156">这介绍了输入和控制需要时，眼睛的双重角色。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-156">This describes the double role of eye-gaze when it's required both for input and control.</span></span> 

- <span data-ttu-id="b8eb7-157">**在单击前保留：** 对于快速目标选择，研究表明，用户的眼睛可以在结束手动单击 (例如，单击) 。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-157">**Leave before click:** For quick target selections, research has shown that a user's eye-gaze can move on before concluding a manual click (for example, an air tap).</span></span> <span data-ttu-id="b8eb7-158">请特别注意，将速度缓慢的控制输入与慢速控制输入 (例如，语音、动手、控制器) 同步。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-158">Pay special attention to synchronizing the fast eye-gaze signal with slower control input (for example, voice, hands, controller).</span></span>

- <span data-ttu-id="b8eb7-159">**小型目标：** 当您尝试读取只是太小而无法阅读的文本时，您是否知道这种感觉？</span><span class="sxs-lookup"><span data-stu-id="b8eb7-159">**Small targets:** Do you know the feeling when you try to read text that is just a bit too small to read comfortably?</span></span> <span data-ttu-id="b8eb7-160">这种令人厌烦的感觉会使您感到厌倦和磨损，因为您尝试重新调整您的眼睛，使其更好地专注。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-160">This straining feeling on your eyes can cause you to feel tired and worn out, because you try to readjust your eyes to focus better.</span></span>
<span data-ttu-id="b8eb7-161">当你强制用户使用目视目标选择在你的应用程序中太小的目标时，你可以在用户中调用这种感觉。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-161">This is a feeling you might invoke in your users when forcing them to select targets that are too small in your application using eye targeting.</span></span>
<span data-ttu-id="b8eb7-162">在设计方面，为了给用户建立一种愉悦舒适的体验，我们建议目标至少在 2° 的视角范围内，最好是再大一些。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-162">For your design, to create a pleasant and comfortable experience for your users, we recommend that targets should be at least 2° in visual angle, preferably larger.</span></span>

- <span data-ttu-id="b8eb7-163">**眼睛不规则的运动** 我们的眼睛执行从固定到固定的快速移动。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-163">**Ragged eye-gaze movements** Our eyes perform rapid movements from fixation to fixation.</span></span> <span data-ttu-id="b8eb7-164">观察录制的眼部运动扫描路径时你会发现，这些路径看起来是不规则的。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-164">If you look at scan paths of recorded eye movements, you can see that they look ragged.</span></span> <span data-ttu-id="b8eb7-165">您的眼睛会在与 *打印头* 或 *手间运动* 比较的情况中快速跳转。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-165">Your eyes move quickly and in spontaneous jumps in comparison to *head-gaze* or *hand motions*.</span></span>  

- <span data-ttu-id="b8eb7-166">**跟踪可靠性：** 当眼睛调整到新的条件时，眼睛跟踪准确性可能会降低变化的光线。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-166">**Tracking reliability:** Eye tracking accuracy may degrade a little in changing light as your eyes adjust to the new conditions.</span></span>
<span data-ttu-id="b8eb7-167">尽管这不会影响应用程序的设计，因为准确性应该在2°限制范围内，因此，用户可能需要重新校准。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-167">While this shouldn't necessarily affect your application design, as the accuracy should be within the 2° limitation, it may be necessary for the user to calibrate again.</span></span> 


## <a name="design-recommendations"></a><span data-ttu-id="b8eb7-168">设计建议</span><span class="sxs-lookup"><span data-stu-id="b8eb7-168">Design recommendations</span></span>
<span data-ttu-id="b8eb7-169">下面是基于目视输入的所述优点和挑战的特定设计建议列表：</span><span class="sxs-lookup"><span data-stu-id="b8eb7-169">The following is a list of specific design recommendations based on the described advantages and challenges for eye-gaze input:</span></span>

1. <span data-ttu-id="b8eb7-170">**眼睛看不到打印头：**</span><span class="sxs-lookup"><span data-stu-id="b8eb7-170">**Eye-gaze isn't the same as Head-gaze:**</span></span>
    - <span data-ttu-id="b8eb7-171">**考虑快速但不规则眼部运动是否适合输入任务：** 虽然快速不规则眼睛运动非常适用于快速选择整个视场中的目标，但它不太适用于需要平滑输入轨迹的任务 (例如，绘制或围绕) 。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-171">**Consider whether fast yet ragged eye movements fit your input task:** While our fast and ragged eye movements are great at quickly selecting targets across our field of view, it's less applicable for tasks that require smooth input trajectories (for example, drawing or encircling annotations).</span></span> <span data-ttu-id="b8eb7-172">在这种情况下，应该首选手部或头部指向。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-172">In this case, hand or head pointing should be preferred.</span></span>
  
    - <span data-ttu-id="b8eb7-173">**避免将某些内容直接附加到用户的视线，例如 (滑块或光标) 。**</span><span class="sxs-lookup"><span data-stu-id="b8eb7-173">**Avoid attaching something directly to the user’s eye-gaze (for example, a slider or cursor).**</span></span>
<span data-ttu-id="b8eb7-174">使用光标时，由于预测眼睛凝视信号中略有偏移，这可能会导致"光标闪烁"效果。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-174">With cursors, this may result in a "fleeing cursor" effect because of slight offsets in the projected eye-gaze signal.</span></span> <span data-ttu-id="b8eb7-175">使用滑块时，它可能会与双角色冲突：用眼睛控制滑块，同时还要检查对象是否位于正确的位置。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-175">With a slider, it can conflict with the double role of controlling the slider with your eyes while also wanting to check whether the object is at the correct location.</span></span> <span data-ttu-id="b8eb7-176">对于滑块的示例，将眼睛凝视与手势结合使用更有意义。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-176">For the example of the slider, it makes more sense to use eye-gaze in combination with hand gestures.</span></span> <span data-ttu-id="b8eb7-177">这意味着，用户可以快速轻松地在许多滑块间切换，提升手部并捏合手指和手指来抓取和移动滑块。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-177">This means that the user could quickly and effortlessly switch among many sliders, raising up their hand and pinching their thumb and index finger to grab and move it.</span></span> <span data-ttu-id="b8eb7-178">释放收缩时，滑块停止移动。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-178">When the pinch is released, the slider stops moving.</span></span> <span data-ttu-id="b8eb7-179">用户可能会感到重负和分散注意力，尤其是在信号不精确时。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-179">Users could become overwhelmed and distracted, especially if the signal is imprecise for that user.</span></span> 
  
2. <span data-ttu-id="b8eb7-180">**将眼睛凝视与其他输入相结合：** 眼动跟踪与其他输入（如手势、语音命令或按钮按下）的集成提供以下优势：</span><span class="sxs-lookup"><span data-stu-id="b8eb7-180">**Combine eye-gaze with other inputs:** The integration of eye tracking with other inputs, such as hand gestures, voice commands or button presses, provides several advantages:</span></span>
    - <span data-ttu-id="b8eb7-181">**允许免费观察：** 鉴于我们眼睛的主要角色是观察环境，因此，必须允许用户四处查看，而不触发任何 (视觉对象、审核等) 反馈或操作。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-181">**Allow for free observation:** Given that the main role of our eyes is to observe our environment, it's important that users are allowed to look around without triggering any (visual, auditory, and so on) feedback or actions.</span></span> 
    <span data-ttu-id="b8eb7-182">将眼动跟踪与另一个输入控件相结合，可以在眼动跟踪观察和输入控制模式之间顺利转换。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-182">Combining eye tracking with another input control allows smooth transitioning between eye tracking observation and input control modes.</span></span>
  
    - <span data-ttu-id="b8eb7-183">**功能强大的上下文提供程序：** 使用有关用户在说语音命令时正在查看什么位置和内容的信息，或者使用手势，可以跨视场无缝地通道输入。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-183">**Powerful context provider:** Using information about where and what the user is looking at while saying a voice command or using a hand gesture allows seamlessly channeling the input across the field-of-view.</span></span> <span data-ttu-id="b8eb7-184">例如：说 _"将其放在此处"_ ，可以通过查看目标及其预期目标来流畅地选择并在场景中放置一个全息影像。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-184">For example: Say _"put that there"_ to quickly and fluently select and position a hologram across the scene by looking at a target and its intended destination.</span></span> 

    - <span data-ttu-id="b8eb7-185">**同步各模输入需要：** 将快速目视运动与更复杂的输入（例如长时间的语音命令或手势）相结合，可以在完成和识别额外的输入命令之前，满足用户已继续查找的风险。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-185">**Need for synchronizing multimodal inputs:** Combining rapid eye movements with more complex inputs, such as long voice commands or hand gestures, bears the risk that the user already continues to look around before the extra input command is finished and recognized.</span></span> <span data-ttu-id="b8eb7-186">如果创建自己的输入控件 (例如，) 自定义手势，请确保记录此输入或大致持续时间的开始，以将其与用户过去查看的内容相关联。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-186">If you create your own input controls (for example, custom hand gestures), make sure to log the onset of this input or approximate duration to correlate it with what a user had looked at in the past.</span></span>
    
3. <span data-ttu-id="b8eb7-187">**目视跟踪输入的细微反馈：** 当查看目标以指示系统正在按预期工作但应保持微妙时，提供反馈很有用。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-187">**Subtle feedback for eye tracking input:** It's useful to provide feedback when a target is looked at to indicate that the system is working as intended but should be kept subtle.</span></span> <span data-ttu-id="b8eb7-188">这可能包括缓慢的混合和缩小、视觉对象突出显示或执行其他微妙目标行为（如缓慢运动），如稍微增加目标大小。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-188">This can include slowly blending in and out, visual highlights, or doing other subtle target behaviors, such as slow motions, such as slightly increasing the target size.</span></span> <span data-ttu-id="b8eb7-189">这表示系统正确检测到用户正在查看目标，而不会中断用户的当前工作流。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-189">This indicates that the system correctly detected that the user is looking at a target without unnecessarily interrupting the user’s current workflow.</span></span> 

4. <span data-ttu-id="b8eb7-190">**避免将非自然目视运动作为输入进行：** 不要强制用户使用特定的眼睛运动 (注视手势) 在应用程序中触发操作。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-190">**Avoid enforcing unnatural eye movements as input:** Don't force users to use specific eye movements (gaze gestures) to trigger actions in your application.</span></span>

5. <span data-ttu-id="b8eb7-191">**帐户 imprecisions：** 我们区分这两种类型的 imprecisions，这些类型对用户很明显：偏移和抖动。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-191">**Account for imprecisions:** We distinguish two types of imprecisions, which are noticeable to users: offset and jitter.</span></span> <span data-ttu-id="b8eb7-192">解决偏移量的最简单方法是提供足够大的目标来与进行交互。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-192">The easiest way to address an offset is to provide sufficiently large targets to interact with.</span></span> <span data-ttu-id="b8eb7-193">建议使用大于2°的视觉角度作为参考。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-193">It's suggested that you use a visual angle greater than 2° as a reference.</span></span> <span data-ttu-id="b8eb7-194">例如，当您拉伸 arm 时，缩略图约为2°。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-194">For instance, your thumbnail is about 2° in visual angle when you stretch out your arm.</span></span> <span data-ttu-id="b8eb7-195">因此，我们的指导如下：</span><span class="sxs-lookup"><span data-stu-id="b8eb7-195">This leads to the following guidance:</span></span>
    - <span data-ttu-id="b8eb7-196">不要强制用户选择 "小目标"。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-196">Don't force users to select tiny targets.</span></span> <span data-ttu-id="b8eb7-197">研究表明，如果目标非常大且系统设计良好，则用户会将其交互描述为轻松、神奇。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-197">Research has shown that if targets are sufficiently large and system are designed well, users describe their interactions as effortless, and magical.</span></span> <span data-ttu-id="b8eb7-198">如果目标太小，则用户就会将体验描述为费力、令人沮丧。</span><span class="sxs-lookup"><span data-stu-id="b8eb7-198">If targets become too small, users describe the experience as fatiguing and frustrating.</span></span>
