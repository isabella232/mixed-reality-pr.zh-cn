<span data-ttu-id="77902-136">**视觉对象复杂性和辅助功能**</span><span class="sxs-lookup"><span data-stu-id="77902-136">**Visual complexity and accessibility**</span></span>

<span data-ttu-id="77902-137">由于全息对象与物理环境混合，透明或半透明窗口上的内容或 UI 清晰度可能会下降。</span><span class="sxs-lookup"><span data-stu-id="77902-137">Since holographic objects blend with the physical environment, content or UI legibility on transparent or translucent windows could be degraded.</span></span> <span data-ttu-id="77902-138">此外，当透明全息对象重叠在一起时，这可能会使用户难以在不太深入的情况下进行交互。</span><span class="sxs-lookup"><span data-stu-id="77902-138">Additionally, when transparent holographic objects are overlaid on top of each other, it could make it difficult for the user to interact because of the confusing depth.</span></span>

<span data-ttu-id="77902-139">**“性能”**</span><span class="sxs-lookup"><span data-stu-id="77902-139">**Performance**</span></span>

<span data-ttu-id="77902-140">若要正确呈现透明或半透明对象，必须将其与存在于后台的任何对象进行排序和混合。</span><span class="sxs-lookup"><span data-stu-id="77902-140">For transparent or translucent objects to render correctly they must be sorted and blended with any objects, which exist in the background.</span></span> <span data-ttu-id="77902-141">透明对象的排序具有适度的 CPU 开销，混合具有相当大的 GPU 开销，因为它不允许 GPU 通过 z 剔除来执行隐藏的图面删除 (例如</span><span class="sxs-lookup"><span data-stu-id="77902-141">Sorting of transparent objects has a modest CPU cost, blending has considerable GPU cost because it doesn't allow the GPU to do hidden surface removal via z-culling (i.e</span></span> <span data-ttu-id="77902-142">深度测试) 。</span><span class="sxs-lookup"><span data-stu-id="77902-142">depth testing).</span></span> <span data-ttu-id="77902-143">不允许隐藏的图面会增加最终呈现的像素所需的操作的数目。</span><span class="sxs-lookup"><span data-stu-id="77902-143">Not allowing hidden surface removal increases the number of operations needed for the final rendered pixel.</span></span> <span data-ttu-id="77902-144">这将导致更多压力填充率约束。</span><span class="sxs-lookup"><span data-stu-id="77902-144">This puts on more pressure fill rate constraints.</span></span>

<span data-ttu-id="77902-145">**LSR 技术的全息影像稳定性问题**</span><span class="sxs-lookup"><span data-stu-id="77902-145">**Hologram stability issue with Depth LSR technology**</span></span>

<span data-ttu-id="77902-146">若要改善全息 reprojection 或全息图稳定性，应用程序可以为每个呈现的帧提交系统的深度缓冲区。</span><span class="sxs-lookup"><span data-stu-id="77902-146">To improve holographic reprojection, or hologram stability, an application can submit a depth buffer to the system for every rendered frame.</span></span> <span data-ttu-id="77902-147">使用 reprojection 的深度缓冲区时，需要为每个颜色呈现的像素写入一个相应深度的深度缓冲区。</span><span class="sxs-lookup"><span data-stu-id="77902-147">When using the depth buffer for reprojection, you need to write a depth buffer for every color rendered pixel a corresponding depth.</span></span> <span data-ttu-id="77902-148">具有深度值的任何像素还应具有颜色值。</span><span class="sxs-lookup"><span data-stu-id="77902-148">Any pixel with a depth value should also have color value.</span></span> <span data-ttu-id="77902-149">如果未遵循上述指导原则，呈现图像中缺少有效深度信息的区域可能会 reprojected 生成项目的方式，这通常作为类似于波形的失真显示。</span><span class="sxs-lookup"><span data-stu-id="77902-149">If the above guidance isn't followed, areas of the rendered image that lack valid depth information may be reprojected in a way that produces artifacts, which are often visible as a wave-like distortion.</span></span>


## <a name="design-guidelines-for-transparent-elements"></a><span data-ttu-id="77902-150">透明元素的设计准则</span><span class="sxs-lookup"><span data-stu-id="77902-150">Design guidelines for transparent elements</span></span>

<span data-ttu-id="77902-151">**使用不透明的 UI 背景**</span><span class="sxs-lookup"><span data-stu-id="77902-151">**Use opaque UI background**</span></span>

<span data-ttu-id="77902-152">默认情况下，透明或半透明对象不会写入深度以允许进行适当的混合。</span><span class="sxs-lookup"><span data-stu-id="77902-152">By default, transparent or translucent objects don't write depth to allow for proper blending.</span></span> <span data-ttu-id="77902-153">缓解此问题的方法包括：使用不透明对象，并确保半透明对象接近于不透明的对象 (例如，在不透明的 backplate) 前面添加半透明的按钮、强制半透明对象写入深度 (不适用于所有方案) ，或呈现代理对象，这些对象只在帧末尾提供深度值。</span><span class="sxs-lookup"><span data-stu-id="77902-153">Ways to mitigate this issue include, using opaque objects, ensuring translucent objects appear close to opaque objects (such as a translucent button in front of an opaque backplate), forcing translucent objects to write depth (not applicable in all scenarios), or rendering proxy objects, which only contribute depth values at the end of the frame.</span></span>

<span data-ttu-id="77902-154">MRTK 中的解决方案-Unity： https://docs.microsoft.com/windows/mixed-reality/mrtk-unity/performance/hologram-stabilization#depth-buffer-sharing-in-unity</span><span class="sxs-lookup"><span data-stu-id="77902-154">Solutions within MRTK-Unity: https://docs.microsoft.com/windows/mixed-reality/mrtk-unity/performance/hologram-stabilization#depth-buffer-sharing-in-unity</span></span>  

<span data-ttu-id="77902-155">通过使用纯和不透明的 backplate，我们可以确保可读性和交互置信度。</span><span class="sxs-lookup"><span data-stu-id="77902-155">By using a solid and opaque backplate, we can secure legibility and interaction confidence.</span></span>

<span data-ttu-id="77902-156">**最大程度地减少受影响的像素数**</span><span class="sxs-lookup"><span data-stu-id="77902-156">**Minimize the number of pixels affected**</span></span>

<span data-ttu-id="77902-157">如果项目必须使用透明对象，请尝试最大程度地减少受影响的像素数。</span><span class="sxs-lookup"><span data-stu-id="77902-157">If your project must use transparent objects, try to minimize the number of pixels affected.</span></span> <span data-ttu-id="77902-158">例如，如果某一对象仅在某些条件下可见 (如) 的加法光亮效果，则在该对象完全不可见时禁用该对象 (而不是将附加颜色设置为黑色) 。</span><span class="sxs-lookup"><span data-stu-id="77902-158">For example, if an object is only visible under certain conditions (like an additive glow effect), disable the object when it's fully invisible (instead of setting the additive color to black).</span></span> <span data-ttu-id="77902-159">对于使用带有 alpha 掩码的四核创建的简单二维形状，请考虑改为使用不透明着色器来创建形状的网格表示形式。</span><span class="sxs-lookup"><span data-stu-id="77902-159">For simple 2D shapes created using a quad with an alpha mask, consider creating a mesh representation of the shape with an opaque shader instead.</span></span>
