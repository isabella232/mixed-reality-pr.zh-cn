# <a name="holographic-frame"></a>全息帧

用户通过其头戴显示设备支持的矩形视区来看到混合现实的世界。 在 HoloLens 上，此矩形区域称为全息框，它可让用户看到覆盖在他们周围现实世界上的数字内容。 针对全息框架进行了优化的设计体验带来了机会，可减轻挑战，并增强了混合现实应用程序的用户体验。

## <a name="designing-for-content"></a>内容设计

通常，设计人员认为需要将他们的体验范围限制为用户可以立即看到的内容，从而牺牲现实世界规模，以确保用户能够完全查看对象。 与复杂应用程序类似的设计器通常会重载全息帧，其中包含内容、难于交互的用户和混乱的界面。 创建混合现实内容的设计器不需要将体验限制为直接在用户面前，并在其即时视图中。 如果已映射用户的物理世界，则所有这些曲面都应被视为数字内容和交互的潜在画布。 在体验中正确设计交互和内容应鼓励用户在其空间中移动、将注意力引向重要内容并帮助查看混合现实的全部潜能。

在应用程序中鼓励移动和浏览的最重要技术可能是 **让用户调整体验**。 使用设备为用户提供短时间的 "任务免费"。 这可以很简单，只是将一个对象置于空间中，并让用户四处移动或 narrating 体验。 这一次应该不会有任何关键任务，也不应像点击那样的特定手势。 目的是让用户在需要交互性或通过应用的进展之前，使用户能够通过设备查看内容。 这一点对于第一次使用的用户尤其重要，因为他们可以通过全息帧和全息影像的性质轻松查看内容。

### <a name="large-objects"></a>大型对象

通常，体验需要的内容（尤其是真实内容）将大于全息帧。 首次引入全息帧时，通常无法容纳在全息帧内的对象应收缩 (以较小的比例或以较小的距离) 。 关键在于让用户在缩放 **使** 帧无法承受之前查看对象的完整大小。 例如，应显示全息影像，以完全适应帧。 这允许用户在将动物的大小调整为靠近用户的实际规模之前，对动物的整体形状 [形成空间理解](scale.md) 。

考虑对象的完整大小后，用户期望在何处移动并查找该对象的特定部分。 在沉浸式内容体验中，通过一种方法引用该内容的完整大小会有所帮助。 例如，如果体验涉及在虚拟房屋的模型周围移动，则使用较小的房屋大小版本来查明它们位于房屋内部的位置会有所帮助。

有关针对大型对象进行设计的示例，请参阅[为大型对象设计。](holographic-frame.md#volvo-cars)

### <a name="many-objects"></a>许多对象

许多对象或组件的体验应考虑使用用户周围的完整空间，以避免直接在用户前面混乱全息帧。 建议将内容引入体验的速度变慢，尤其是对于计划向用户提供许多对象的体验。 关键在于让用户 **了解** 体验中的内容布局，这有助于用户在空间上了解内容更新时周围的内容。

实现此目的的一种技术是提供持久点 (在将内容定位到现实世界的体验) 称为特征点。 例如，路标可以是实际的物理对象（例如显示数字内容的表），也可以是数字对象，例如内容经常出现在其中的数字屏幕集。 还可将对象放入全息帧的绕，以鼓励用户查看关键内容。 绕之外的内容发现可由 "注意" [导演](holographic-frame.md#attention-directors)进行干预。

将对象放置在绕中可鼓励用户寻找一侧，这可由 "注意" 导演进行干预，如下所述。 有关全息帧注意事项的详细信息，请参阅 [舒适](comfort.md#holographic-frame-considerations) 的文档。

## <a name="interaction-considerations"></a>交互注意事项

与内容一样，混合现实体验中的交互不需要限于用户可以立即看到的内容。 交互可以在用户周围的实际空间中的任何位置发生。 这些交互可帮助鼓励用户四处移动和探索经验。

### <a name="attention-directors"></a>注意控制器

指示兴趣点或关键交互对于使用户获得经验非常重要。 用户注意和巧妙帧的移动可通过微妙或重型方式定向。 请记住在混合现实中平衡密切的主管， (尤其是在开始体验) ，以避免用户不堪重负。 通常，有两种类型的注意控制器：
* **视觉对象控制器：** 让用户知道他们应该在特定方向上移动的最简单方法是提供直观的指示。 这可以通过视觉效果来实现 (例如，用户可在此过程中以可视方式跟踪) ，甚至是简单的方向箭头的路径。 任何可视指示器都应在用户环境中进行接地，而不是 "附加" 到全息帧或光标。
* **音频控制器：** [空间音效](spatial-sound-design.md) 可提供一种功能强大的方式在场景中建立对象。 可以提醒用户进入体验的对象，或将用户的视图移动到关键对象，将焦点直接指向空间中的特定点。 与视觉主管相比，使用音频控制器引导用户关注可能更细微且更具侵入性。 在某些情况下，最好从音频目录开始，如果用户无法识别提示，则继续访问视觉对象目录。 音频控制器还可以与视觉控制器配对，以增加重点。

### <a name="commanding-navigation-and-menus"></a>命令、导航和菜单

理想情况下，混合现实体验中的接口与它们控制的数字内容紧密配对。 因此，自由浮动的 2D 菜单通常不适合于交互，并且对于在全息帧内操作过于轻松的用户来说可能很困难。 对于需要接口元素（如菜单或文本字段）的体验，请考虑使用 [标记](billboarding-and-tag-along.md) 沿方法在短暂延迟后跟随全息帧。 避免将内容锁定到帧（如向上显示）中，因为这样做可能会使用户无法理解，并破坏场景中其他数字对象的沉浸感。

还可以将接口元素直接放在它们控制的特定内容上，从而允许交互自然地围绕用户的物理空间进行。 例如，将复杂菜单分解为单独的部分，将每个按钮或一组控件附加到交互影响的特定对象。 若要进一步理解此概念，请考虑使用 [可交互对象](interactable-object.md)。

### <a name="gaze-and-gaze-targeting"></a>凝视和凝视目标

全息帧为开发人员提供了一个工具，用于触发交互并评估用户注意力的停留位置。 凝[视是](gaze-and-commit.md) [HoloLens](interaction-fundamentals.md)上的关键交互之一，其中凝视可以与手势[ (（](gaze-and-commit.md#composite-gestures)例如，通过声点击) 或语音[ (](voice-input.md)进行更短、更自然的基于语音的交互) 。 因此，这会使全息帧成为观察数字内容并与之交互的空间。 如果体验在用户的空间周围与多个对象进行交互 (例如，通过 "注视 + 手势") 多选对象，请考虑将这些对象置于用户的视图中，或限制必要的头运动量来改善 [用户舒适](comfort.md)。

注视还可用于跟踪用户体验，并查看用户对哪些对象或部分用户最关注的部分。 这会特别用于调试体验，允许使用分析工具（如热图）查看用户花费最多时间或缺少特定对象或交互的位置。 注视跟踪还可以为 facilitators 提供强大的工具， (参见 [Lowe 的厨房](holographic-frame.md#lowes-kitchen) 示例) 。

## <a name="performance"></a>性能

正确使用全息帧是 [性能质量](../develop/platform-capabilities-and-apis/understanding-performance-for-mixed-reality.md) 体验的基础。 常见的技术 (和可用性) 难题是使用数字内容使用户的帧超载，从而导致渲染性能下降。 请考虑对用户使用整个空间来排列数字内容（使用上述方法），以减少呈现的负担并确保最佳显示质量。

HoloLens 全息帧中的数字内容也可以与 [稳定平面](../develop/platform-capabilities-and-apis/case-study-using-the-stabilization-plane-to-reduce-holographic-turbulence.md) 配对，以实现最佳性能和 [全息影像稳定性](../develop/platform-capabilities-and-apis/hologram-stability.md)。

## <a name="examples"></a>示例

### <a name="volvo-cars"></a>Volvo 汽车

<iframe width="940" height="530" src="https://www.youtube.com/embed/DilzwF90vec" frameborder="0" allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

在 Volvo 汽车的 showroom 体验中，客户受邀了解一项新汽车在 Volvo 关联的中的功能。 Volvo 面临着一个与全息框架有关的挑战：全尺寸汽车太大，无法放在用户旁。 解决方案是开始使用物理地标（一个中央表）的体验，将较小的汽车数字模型放置在表的顶部。 这可确保用户在引入时看到完整汽车，从而在以后的体验中，一旦汽车增长到实际规模，就能够理解空间。

其体验还利用视觉元素，从表上的小型汽车模型到展示房间中的墙，创建长视觉效果。 这会导致"幻窗口"效果，显示距离上汽车的完整视图，以真实规模展示汽车的更多特征。 头部移动是水平移动，用户无需任何直接交互 (而是以可视方式收集提示，以及从用户对体验和) 。

### <a name="lowes-kitchen"></a>Lowe's Kitchen

Lowe 的商店体验邀请客户进入一个全面规模的厨房模拟，以展示通过 HoloLens 看到的各种重新建模机会。 商店中的厨房为数字对象提供物理背景，提供一个空白的画布，包含设备、副台和机箱，使混合现实体验能够展开。

物理图面充当静态地标，让用户在体验中自行地面，因为 Lowe 的关联人员会引导用户完成不同的产品选项并完成操作。 这样一来，关联人就可以以语言方式将用户注意力引导到"厨房"或"厨房中心"，以展示数字内容。

![Lowe 的关联公司使用平板电脑来指导客户完成 HoloLens 体验。](images/loweskitchen-750px.jpg)<br>
*Lowe 的关联公司使用平板电脑来指导客户完成 HoloLens 体验。*

用户体验部分由 Lowe 关联控制的平板电脑体验管理。 在这种情况下，关联人员的角色之一是限制头部过度移动，将焦点顺利引导到厨房中的兴趣点。 Tablet 体验还提供 Lowe 与厨房的热度地图视图形式的 "注视" 数据的关联，有助于了解用户的 dwelling (例如，在 cabinetry) 的特定区域，以便更准确地为其提供装修指导。

若要深入了解 Lowe 的厨房体验，请参阅 [Microsoft 在 Ignite 2016 上的主题](https://www.youtube.com/watch?v=gC_4JxF0e_k)。
